为什么要对 Gradle 统一管理

在实际项目中，经常会出现一个 Project 中有多个 Module 的情况，如果每个 Module 又同时拥有相同的依赖，比如 v4 包，那么，每次 v4 包升级的时候修改多个 Module 中的 build.gradle 文件，更新其版本都是一件麻烦的事情，而且很有可能忘记某个 Module。所以，将依赖进行统一管理，对于一个 Project 下拥有多个 Module 的情况来说是非常必要的。
